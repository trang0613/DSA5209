Problem 3¶

Estimate $I = \int_{\mathbb{R}^2} -\log|x-y|\,\psi(x)\psi(y)\,dx\,dy$ where $\psi = \mathcal{N}(0,\sigma^2)$, $\sigma=1/4$. True value $I\approx 1.6749$.

1. Basic Monte Carlo¶

Since $\psi(x)\psi(y)$ is the joint density, draw $x,y\sim\mathcal{N}(0,\sigma^2)$ and average $-\log|x-y|$:

$$I_N = \frac{1}{N}\sum_{i=1}^N -\log|x_i-y_i|$$

Difficulty: $-\log|u|\to\infty$ as $u\to 0$. Many samples fall near $x\approx y$, inflating variance.

In [2]:
def monte_carlo(N):
    t0 = time.perf_counter()
    x   = np.random.normal(0, sigma, N)
    y   = np.random.normal(0, sigma, N)
    diff = np.abs(x - y)
    vals = np.where(diff > 1e-300, -np.log(diff), 0.0)
    est  = np.mean(vals)
    ms   = (time.perf_counter() - t0) * 1000
    return est, ms

est, t = monte_carlo(100_000)
print(f'MC (N=100k): {est:.4f}  error={abs(est-TRUE_I):.4f}  time={t:.2f}ms')

MC (N=100k): 1.6784  error=0.0035  time=6.25ms

2. Importance Sampling¶

Reformulation¶

Let $u=x-y$. Since $x,y\sim\mathcal{N}(0,\sigma^2)$ independently, $u\sim g=\mathcal{N}(0,2\sigma^2)$, so:

$$I = \mathbb{E}_g[-\log|u|]$$

Why naive MC has high variance¶

The function $h(u)=-\log|u|$ spikes sharply near $u=0$. The Gaussian $g$ has non-negligible mass near 0, so many samples produce very large $h(u_i)$, blowing up the variance.

IS strategy: mixture proposal¶

We want a proposal $q$ that oversamples near $u=0$ (where $h$ is large) relative to $g$. We use a mixture:

$$q(u) = (1-\varepsilon)\,g(u) + \varepsilon\,\text{Uniform}(-\delta,\delta)$$

with $\varepsilon=0.3$, $\delta=0.05$. The uniform component injects extra samples into the dangerous region near 0. Importance weights correct for the tilting:

$$w_i = \frac{g(u_i)}{q(u_i)}, \qquad I_N^{IS} = \frac{\sum_i w_i h(u_i)}{\sum_i w_i}$$

eps   = 0.3
delta = 0.05

def sample_q(N):
    """Draw N samples from the mixture proposal q."""
    n_g = int(N * (1 - eps))
    n_u = N - n_g
    return np.concatenate([
        np.random.normal(0, sigma_u, n_g),
        np.random.uniform(-delta, delta, n_u)
    ])

def log_q_pdf(u):
    """Log-density of the mixture proposal."""
    log_g    = norm.logpdf(u, scale=sigma_u)
    log_unif = np.where(np.abs(u) <= delta,
                        np.log(1/(2*delta)), -np.inf)
    return np.logaddexp(np.log(1-eps) + log_g,
                        np.log(eps)   + log_unif)

def importance_sampling(N):
    t0  = time.perf_counter()
    u   = sample_q(N)
    u   = u[np.abs(u) > 1e-300]           # guard against exact zero
    log_w = norm.logpdf(u, scale=sigma_u) - log_q_pdf(u)
    log_w -= log_w.max()                   # numerical stability
    w   = np.exp(log_w);  w /= w.sum()
    h   = -np.log(np.abs(u))
    est = np.sum(w * h)
    ESS = 1.0 / np.sum(w**2)
    ms  = (time.perf_counter() - t0) * 1000
    return est, ms, ESS

est_is, t_is, ess = importance_sampling(100_000)
print(f'IS  (N=100k): {est_is:.4f}  error={abs(est_is-TRUE_I):.4f}  '
      f'time={t_is:.2f}ms  ESS={ess:.0f} ({100*ess/100_000:.1f}%)')

IS  (N=100k): 1.6757  error=0.0008  time=6.71ms  ESS=76972 (77.0%)

5. Summary¶

Basic MCIS Estimator
Samplerx, y ~ N(0, sigma^2)u ~ (1-eps)*N(0,2sigma^2) + eps*Uniform(-delta,delta)
Difficulty-log|x-y| spikes near x=y, causing high varianceExtra uniform mass near u=0 oversamples the dangerous region
WeightsAll equal (1/N)w_i = g(u_i)/q(u_i), self-normalised
ESSN (full)~70-80% of N, variance reduction ~1.5-2x

Common mistake: drawing u ~ g and applying weights |u|^{-alpha} does NOT produce a valid IS estimator for E_g[h]. Self-normalised IS requires samples from a different proposal q, with weights g/q correcting for the mismatch.
